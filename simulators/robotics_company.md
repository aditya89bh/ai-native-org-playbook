# Robotics Company Simulator

## Organization profile

A robotics company using agents to coordinate product, engineering, operations, customer deployments, incidents, and field learning.

## Departments

- Engineering
- Product
- Operations
- Customer support
- Sales
- Research

## Agent roles

- Incident Agent
- Product Feedback Agent
- Workflow Orchestration Agent
- Knowledge Agent
- Research Agent
- Chief of Staff Agent

## Core workflows

- Pull request review
- Incident response
- Customer onboarding
- Product discovery
- Knowledge management

## Memory map

- Field incident memory
- Customer deployment memory
- Architecture decision memory
- Product feedback memory
- Support ticket memory

## Governance map

- Production robot behavior changes require engineering approval
- Customer deployment commitments require account owner approval
- Safety-related incidents require strict audit
- Product roadmap changes require product approval

## Metrics

- Incident resolution time
- Field learning reuse
- Customer onboarding quality
- Review cycle time
- Product feedback traceability

## Before scenario

Field issues, product feedback, support tickets, and engineering decisions do not reliably feed each other.

## After scenario

Agents connect field memory, incident response, product discovery, and engineering review so the company learns from deployments faster.
