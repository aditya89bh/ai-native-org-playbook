# Agent Governance Stack

The governance stack defines the controls required before agents can operate inside real workflows.

## Layer 1: Identity

Every agent needs a stable identity, owner, purpose, and scope.

## Layer 2: Permissions

Permissions define which tools, systems, records, and actions the agent can access.

## Layer 3: Policy

Policies define what the agent may do, may not do, and must escalate.

## Layer 4: Review

Review defines approval points, sampling rules, and post-action checks.

## Layer 5: Audit

Audit logs preserve inputs, outputs, decisions, tools, approvals, and outcomes.

## Layer 6: Correction

Corrections must update future behavior, not just fix one output.

## Layer 7: Decommissioning

Agents need retirement criteria when workflows change or reliability drops.
