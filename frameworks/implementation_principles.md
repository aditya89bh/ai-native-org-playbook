# Implementation Principles

## 1. Redesign before automation

Do not automate a workflow you do not understand.

## 2. Give agents roles, not vague tasks

Durable roles make evaluation, ownership, and governance possible.

## 3. Keep decision rights explicit

Every agent action should have a known authority level.

## 4. Build memory with forgetting

Memory without retention rules becomes organizational clutter.

## 5. Measure work outcomes

Usage, novelty, and excitement are not sufficient evidence.

## 6. Treat governance as design

Governance is not a blocker. It is how agents earn more authority safely.
