# AI-Native Operating Model

An AI-native operating model defines how work flows through humans, agents, tools, memory, and governance.

## Core components

1. Human roles
2. Agent roles
3. Workflow ownership
4. Decision rights
5. Memory capture
6. Tool permissions
7. Review and escalation
8. Metrics

## Operating model rule

Every workflow must answer four questions:

1. Who observes?
2. Who decides?
3. Who acts?
4. Who remembers?

## Design pattern

Do not automate isolated tasks first. Map the workflow, identify decision points, then assign the right mix of human judgment, agent support, memory, and governance.
