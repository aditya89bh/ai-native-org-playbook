# Customer Support Department Redesign

## Current operating model

Support teams handle customer questions, incidents, complaints, account issues, and product feedback.

## AI-native redesign

AI-native support uses agents for triage, suggested replies, knowledge retrieval, sentiment detection, escalation, and feedback memory.

## Human-led work

- Sensitive complaints
- High-value account issues
- Refund exceptions
- Relationship repair
- Policy exceptions

## Agent-assisted work

- Ticket summarization
- Reply drafting
- Knowledge retrieval
- Sentiment detection
- Escalation recommendation

## Agent-executed work

- Ticket tagging
- Routing
- Duplicate detection
- Status updates
- Knowledge base gap logging

## Recommended agents

- Support Triage Agent
- Reply Drafting Agent
- Knowledge Retrieval Agent
- Escalation Agent
- Customer Memory Agent

## Core workflows

- Ticket triage
- Customer reply
- Escalation handling
- Incident communication
- Knowledge base improvement

## Memory map

- Customer memory
- Ticket memory
- Incident memory
- Policy memory
- Product feedback memory

## Governance

Humans approve sensitive responses, refunds, account-impacting actions, legal-risk language, and escalations involving major customers.

## Metrics

- First response time
- Resolution time
- Escalation accuracy
- Customer satisfaction
- Knowledge reuse
- Human override rate

## Risks

- Incorrect support answers
- Tone mismatch
- Over-automation of angry customers
- Policy hallucination
- Loss of customer context

## Maturity checklist

- Support knowledge base is source-linked
- Escalation rules are explicit
- Sensitive cases remain human-led
- Customer memory has retention rules
- Reply quality is sampled regularly
