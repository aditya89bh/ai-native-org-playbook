# HR Department Redesign

## Current operating model

HR teams coordinate hiring, onboarding, performance, employee relations, learning, policy, and culture through sensitive human workflows.

## AI-native redesign

AI-native HR uses agents for coordination, policy retrieval, onboarding support, learning recommendations, hiring operations, and people analytics while preserving human accountability.

## Human-led work

- Employee relations
- Final hiring decisions
- Sensitive performance conversations
- Culture design
- Conflict resolution

## Agent-assisted work

- Candidate screening support
- Interview packet preparation
- Policy retrieval
- Onboarding planning
- Learning path suggestions

## Agent-executed work

- Scheduling support
- Checklist updates
- Document routing
- Training reminders
- FAQ responses within policy

## Recommended agents

- Hiring Operations Agent
- Onboarding Agent
- Policy Retrieval Agent
- Learning Agent
- People Analytics Agent

## Core workflows

- Hiring pipeline
- Employee onboarding
- Policy support
- Performance review preparation
- Learning and development

## Memory map

- Hiring process memory
- Employee onboarding memory
- Policy memory
- Learning memory
- Performance process memory

## Governance

Humans must approve hiring decisions, performance conclusions, compensation changes, disciplinary actions, and sensitive employee communications.

## Metrics

- Time to hire
- Onboarding completion
- Policy response quality
- Training completion
- Candidate experience
- Employee trust

## Risks

- Bias amplification
- Over-automation of sensitive topics
- Privacy violations
- Unsupported policy interpretation
- Employee distrust

## Maturity checklist

- Sensitive data access is restricted
- Hiring decisions remain human accountable
- Policy sources are traceable
- Employee-facing agents have clear scope
- Bias and privacy reviews are routine
