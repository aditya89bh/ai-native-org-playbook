# Memory by Department

| Department | Most important memory types |
| --- | --- |
| Sales | Account, conversation, proposal, objection, decision |
| Marketing | Campaign, customer signal, positioning, experiment, decision |
| Product | Customer feedback, roadmap, experiment, release, decision |
| Engineering | Architecture, incident, release, review, technical debt |
| Design | Research, component, brand, usability, design decision |
| HR | Hiring, onboarding, policy, learning, performance process |
| Finance | Budget, vendor, forecast, control exception, policy |
| Operations | Process, vendor, incident, SOP, improvement |
| Customer Support | Customer, ticket, incident, policy, product feedback |
| Research | Source, evidence, hypothesis, interview, decision |
| Legal and Compliance | Contract, policy, risk, obligation, decision |

## Rule

Department memory should be designed around reuse. If a memory type does not improve a future decision or workflow, do not capture it by default.
