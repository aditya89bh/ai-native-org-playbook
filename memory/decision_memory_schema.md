# Decision Memory Schema

Decision memory captures why a decision was made, not only what was chosen.

## Fields

- Decision ID
- Decision date
- Owner
- Context
- Options considered
- Selected option
- Rejected options
- Rationale
- Evidence used
- Risks accepted
- Expected outcome
- Review date
- Outcome status

## Use cases

- Product prioritization
- Budget allocation
- Vendor selection
- Architecture decision
- Hiring decision process

## Failure mode

If only the final decision is stored, future teams cannot understand the tradeoff and may repeat the same debate.
