# Meeting Memory Schema

Meeting memory captures decisions, commitments, unresolved questions, and context worth reusing.

## Fields

- Meeting ID
- Date
- Participants
- Purpose
- Decisions made
- Action items
- Owners
- Deadlines
- Open questions
- Sensitive notes
- Follow-up date
- Source transcript or notes

## What not to capture

- Casual comments
- Unnecessary personal context
- Unverified claims
- Sensitive information without purpose

## Design rule

A meeting summary is not enough. The memory should preserve decisions, commitments, and follow-up obligations.
