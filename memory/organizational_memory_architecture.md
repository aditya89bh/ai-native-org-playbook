# Organizational Memory Architecture

AI-native organizations need active memory, not just storage.

## Memory layers

1. Decision memory
2. Meeting memory
3. Project memory
4. Customer memory
5. Policy memory
6. Incident memory
7. Learning memory

## Memory event schema

- Event ID
- Timestamp
- Source
- Actor
- Context
- Decision or observation
- Confidence
- Retention policy
- Retrieval tags
- Correction history

## Design rules

- Capture only what can be reused.
- Separate facts, decisions, assumptions, and opinions.
- Store confidence and source.
- Expire memory intentionally.
- Make corrections first-class events.
