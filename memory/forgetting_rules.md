# Forgetting Rules

Forgetting is an intentional design choice, not a failure of memory.

## When to forget

- Memory is outdated
- Memory is inaccurate
- Memory is sensitive and no longer needed
- Memory creates unfair bias
- Memory is duplicated
- Memory has no future decision value

## Forgetting modes

| Mode | Meaning |
| --- | --- |
| Expire | Stop retrieval after a date |
| Archive | Preserve for audit but remove from normal retrieval |
| Redact | Remove sensitive fields |
| Supersede | Prefer newer corrected memory |
| Delete | Remove when policy requires it |

## Design rule

Agents should know not only what to remember, but when memory should stop influencing work.
