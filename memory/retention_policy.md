# Retention Policy

Retention policies define how long memory should influence work.

## Retention tiers

| Tier | Use | Default |
| --- | --- | --- |
| Ephemeral | Short-term coordination | 7 to 30 days |
| Operational | Workflow execution | 30 to 180 days |
| Project | Project history | Project lifetime plus review |
| Decision | Durable rationale | Long-term |
| Policy | Rules and constraints | Until superseded |
| Incident | Failure learning | Long-term with review |
| Sensitive | Personal or restricted context | Minimized and tightly scoped |

## Review questions

- Does this memory still improve decisions?
- Is it still accurate?
- Is it still allowed to be used?
- Has it become harmful or misleading?
- Who owns correction or deletion?
