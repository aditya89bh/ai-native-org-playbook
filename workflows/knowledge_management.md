# Knowledge Management Workflow

## Current workflow

Knowledge management often depends on scattered documents, stale SOPs, inconsistent tagging, and hard-to-reuse decisions.

## AI-native redesign

Agents support knowledge retrieval, stale-document detection, update suggestions, source linking, and missing-document alerts.

## Human steps

- Own canonical knowledge
- Approve policy and SOP changes
- Resolve conflicting guidance
- Define knowledge standards

## Agent steps

- Retrieve relevant docs
- Flag outdated content
- Suggest updates
- Link related decisions
- Identify gaps

## Shared steps

- SOP review
- Knowledge base cleanup
- Decision documentation
- Policy update review

## Memory events

- Knowledge asset created
- Source linked
- Update suggested
- Owner approval recorded
- Retention rule applied

## Governance points

Humans approve canonical content, policy changes, sensitive material, and high-impact process guidance.

## Metrics

- Retrieval relevance
- Source freshness
- Duplicate reduction
- Update acceptance rate
- Knowledge reuse

## Risks

- Stale answers
- Knowledge clutter
- Unsupported guidance
- Poor access control
