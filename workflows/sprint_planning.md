# Sprint Planning Workflow

## Current workflow

Sprint planning converts product priorities, engineering capacity, technical debt, and dependencies into a near-term execution plan.

## AI-native redesign

Agents support backlog summarization, dependency detection, capacity notes, risk flags, and sprint memory while humans own prioritization and commitments.

## Human steps

- Prioritize work
- Resolve tradeoffs
- Commit capacity
- Approve sprint scope
- Own delivery judgment

## Agent steps

- Summarize backlog items
- Retrieve related decisions
- Flag dependencies
- Draft sprint plan
- Update planning memory

## Shared steps

- Backlog refinement
- Risk review
- Scope adjustment
- Sprint retrospective

## Memory events

- Sprint goal logged
- Scope rationale recorded
- Dependency captured
- Risk noted
- Retrospective lesson saved

## Governance points

Humans approve sprint commitment, customer-impacting work, high-risk technical changes, and scope changes.

## Metrics

- Planning time
- Scope churn
- Dependency miss rate
- Sprint completion rate
- Retrospective reuse

## Risks

- Unrealistic planning
- Missed dependencies
- Over-reliance on stale estimates
- Weak team ownership
