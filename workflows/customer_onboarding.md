# Customer Onboarding Workflow

## Current workflow

Customer onboarding usually includes kickoff, requirements gathering, setup, training, success planning, risk tracking, and handoff to steady-state support.

## AI-native redesign

Agents support onboarding plans, context retrieval, task tracking, documentation drafts, risk detection, and customer memory creation.

## Human steps

- Build customer trust
- Align expectations
- Resolve sensitive blockers
- Approve success plan
- Manage executive relationships

## Agent steps

- Draft onboarding plan
- Track checklist completion
- Summarize kickoff notes
- Retrieve similar customer patterns
- Flag unresolved risks

## Shared steps

- Kickoff preparation
- Success plan creation
- Training coordination
- Handoff review

## Memory events

- Customer goals captured
- Requirements recorded
- Risk notes updated
- Success plan logged
- Handoff memory created

## Governance points

Humans approve customer commitments, timeline changes, commercial exceptions, and sensitive escalations.

## Metrics

- Time to value
- Checklist completion
- Risk detection rate
- Handoff quality
- Customer satisfaction

## Risks

- Misstated commitments
- Incomplete context
- Poor handoff
- Missed customer risks
