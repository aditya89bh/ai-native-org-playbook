# Memory and Governance Depth

Phase 5 deepens the safety and control layers of the AI-native organization framework.

## Memory coverage

- Memory event schema
- Decision memory schema
- Meeting memory schema
- Retention policy
- Correction workflow
- Forgetting rules
- Access scopes

## Governance coverage

- Permission tiers
- Audit log model
- Approval matrix
- Escalation patterns
- Governance risk scoring

## CLI usage

```bash
ai-org-playbook governance governance/specs/high_risk_action.json
ai-org-playbook memory-policy --tier decision --confidence 0.9
```

## Design rule

Memory without governance creates risk. Governance without memory creates bureaucracy. AI-native organizations need both together.
