# Consulting Guide

This repository can be used as a consulting toolkit for AI-native organization redesign.

## Engagement structure

### 1. Diagnose

Use readiness, maturity, department, workflow, governance, and simulator tools to understand the current operating model.

### 2. Select a pilot

Choose one bounded workflow with visible business value, measurable baseline, and manageable governance risk.

### 3. Design the operating model

Define:

- Human owner
- Agent roles
- Workflow steps
- Memory events
- Governance points
- Success metrics

### 4. Implement carefully

Start with assistive and review-heavy workflows. Increase autonomy only when evidence supports it.

### 5. Review and scale

Use reports, metrics, correction logs, and governance reviews to decide whether to expand, restrict, or redesign.

## Suggested workshop format

1. Current-state workflow mapping
2. Agent opportunity scoring
3. Memory and governance design
4. Pilot selection
5. Roadmap generation

## Consulting warning

Do not sell agent adoption as transformation. The real transformation is workflow redesign, memory architecture, governance, and measurable operating leverage.
