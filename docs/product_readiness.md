# Product Readiness Checklist

Use this checklist before a public release.

## Usability

- README explains the repository quickly
- Quickstart can be followed without extra context
- CLI examples are copy-pasteable
- Documentation site builds cleanly
- Examples gallery gives a useful tour

## Engineering

- Tests pass
- Lint passes
- Type checks pass
- CLI commands have smoke examples
- JSON specs are consistent with the docs

## Content quality

- Frameworks are practical
- Department playbooks are specific
- Agent roles include permissions and failure modes
- Workflows include memory and governance
- Simulators connect departments, workflows, agents, and metrics

## Release judgment

Release when someone can clone the repository, run the CLI, read the docs, and use the templates to design one realistic AI-native pilot.
